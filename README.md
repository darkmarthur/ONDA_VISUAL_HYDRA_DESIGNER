# Hydra Visual Editor

**Hydra Visual Editor** is an open-source visual interface built on top of **Hydra Video Synth**, designed to make the creation, exploration, and live execution of visuals more accessible through a node-based workflow.

This tool does not replace Hydra, nor does it try to reinvent it. On the contrary, it was created as a **tribute** and a visual interaction layer on top of the original work by **Olivia Jack**, with the goal of bringing Hydra closer to visual artists, live coders, and especially **live VJs** who need to experiment, improvise, and prototype visuals quickly in performance contexts.

Its workflow is intentionally reminiscent of **TouchDesigner** in order to make the interface feel familiar to artists already used to node-based visual tools, and to make the learning curve more approachable.

---

## What this tool is

Hydra Visual Editor is a visual editor that lets users build Hydra patches using nodes, connections, and editable parameters instead of relying only on handwritten code.

Behind the interface, the real engine is still **Hydra Video Synth**.  
All visuals are generated by Hydra itself; this project simply offers an alternative way to interact with its logic, syntax, and workflow.

---

## Purpose of the project

This project was created to:

- make Hydra easier to learn through a visual interface
- help users understand the relationship between nodes and Hydra code
- facilitate the **learning of Hydra**
- enable **rapid prototyping** of visual projects
- accelerate live experimentation and improvisation
- offer a workflow that feels familiar to users coming from tools like **TouchDesigner**
- serve as a bridge between node-based patching and the actual syntax and behavior of Hydra

It is especially intended for:

- **live VJs**
- generative visual artists
- audiovisual performers
- people who want to learn Hydra in a more accessible way
- artists who enjoy node-based workflows similar to TouchDesigner, but love the power and spirit of Hydra

## Clone and run locally

### 1. Clone the repository

```bash
git clone <REPOSITORY_URL>
cd <PROJECT_FOLDER_NAME>
```

> Replace `<REPOSITORY_URL>` and `<PROJECT_FOLDER_NAME>` with the actual repository information.

### 2. Install dependencies

Using npm:

```bash
npm install
```

Or using yarn:

```bash
yarn install
```

### 3. Run the project locally

Using npm:

```bash
npm run dev
```

Or using yarn:

```bash
yarn dev
```

### 4. Open it in your browser

```txt
http://localhost:3000
```

## Production build

## Project philosophy

Hydra Visual Editor is built around a simple idea:

**make Hydra more accessible, more visible, and more performative, without changing its essence.**

It is not meant to replace pure live coding.  
It is not meant to oversimplify Hydra.  
It is meant to offer another entry point: a visual interface for experimenting, learning, and performing visuals live with more freedom.

Its node-based interaction is intentionally familiar to artists who come from **TouchDesigner-style workflows**, helping them feel at home while learning Hydra more quickly.
